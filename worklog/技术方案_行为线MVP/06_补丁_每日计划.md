# Module 6 — 补丁：每日计划恢复

## 背景

Module 1 的 migration 把 `daily_tasks` 改名为 `anomaly_suggestion`，让用户失去了"写每日计划"这个原有功能。这是设计漏洞：

| 概念 | 语义 | 谁写 |
|---|---|---|
| 每日计划 | 用户写的"今天想做什么"，基线 | 用户 |
| 异常值建议 | AI 在某个时刻的"打断回归"干预 | AI |

两者**不应共用同一张表**。本补丁恢复"每日计划"作为独立特性，与 `AnomalySuggestion` 完全分离。

读本文件前请先看 `00_总览.md` 第 1.2 节"不衡量执行原则" —— 它对本补丁的影响：**AI 永远不读、不写、不评分 daily_plan 表**。`isDone` 是用户自己给自己打勾，不是系统打分。

---

## 1. 实现状态判断（实现 AI 必读）

本补丁默认 **v11 已经部署**到设备上，因此采用 v11 → v12 migration。

**例外情况**：如果 v11 还没真正装到任何设备（仅在开发机器测过），可以**把本文档第 2.3 节的 SQL 合并进 `Migration_10_11.kt`**，不必新建 v12，避免无意义的版本递增。

判断依据：
- 装机的 KHUP app 启动后能看到 `今日`/`历史`/`AI`/`设置` 四 tab → 已部署 v11，走 v12
- 仅在 emulator/开发机有过一次安装，且作者 Kian 同意清除该开发机数据 → 可合并进 v11

**不确定时，默认走 v12**（更安全，不会改已发布的 migration）。

## 2. 数据层

### 2.1 Entity

文件：`core/data/db/entities/DailyPlan.kt`

```kotlin
@Entity(
    tableName = "daily_plan",
    indices = [
        Index("dayStartMs"),
        Index("isDone"),
    ],
)
data class DailyPlan(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val title: String,
    val note: String? = null,           // 可选展开（多行说明）
    val dayStartMs: Long,
    val isDone: Boolean = false,
    val createdAt: Long,
    val completedAt: Long? = null,
    val sortOrder: Int = 0,             // 用户拖拽排序用；初次插入 = createdAt 的秒数
)
```

字段约束：
- `title` 长度 1-100 字符
- `note` 长度 0-500 字符
- 同一 `dayStartMs` 内 `sortOrder` 递增

### 2.2 DAO

文件：`core/data/db/DailyPlanDao.kt`

```kotlin
@Dao
interface DailyPlanDao {

    @Insert
    suspend fun insert(item: DailyPlan): Long

    @Update
    suspend fun update(item: DailyPlan)

    @Delete
    suspend fun delete(item: DailyPlan)

    @Query("SELECT * FROM daily_plan WHERE id = :id")
    suspend fun getById(id: Long): DailyPlan?

    /** 当日所有计划，按 sortOrder + createdAt 排序。 */
    @Query("""
        SELECT * FROM daily_plan
        WHERE dayStartMs = :dayStartMs
        ORDER BY sortOrder ASC, createdAt ASC
    """)
    fun observeByDay(dayStartMs: Long): Flow<List<DailyPlan>>

    @Query("UPDATE daily_plan SET isDone = :isDone, completedAt = :completedAt WHERE id = :id")
    suspend fun setDone(id: Long, isDone: Boolean, completedAt: Long?)

    @Query("UPDATE daily_plan SET title = :title, note = :note WHERE id = :id")
    suspend fun updateContent(id: Long, title: String, note: String?)

    /** 当日完成数 / 总数（仅供 UI 显示用户自己的进度，AI 不消费）。 */
    @Query("SELECT COUNT(*) FROM daily_plan WHERE dayStartMs = :dayStartMs")
    fun observeCountByDay(dayStartMs: Long): Flow<Int>

    @Query("SELECT COUNT(*) FROM daily_plan WHERE dayStartMs = :dayStartMs AND isDone = 1")
    fun observeDoneCountByDay(dayStartMs: Long): Flow<Int>
}
```

### 2.3 Migration_11_12

文件：`core/data/db/migrations/Migration_11_12.kt`

```kotlin
val MIGRATION_11_12 = object : Migration(11, 12) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("""
            CREATE TABLE IF NOT EXISTS daily_plan (
                id INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
                title TEXT NOT NULL,
                note TEXT,
                dayStartMs INTEGER NOT NULL,
                isDone INTEGER NOT NULL DEFAULT 0,
                createdAt INTEGER NOT NULL,
                completedAt INTEGER,
                sortOrder INTEGER NOT NULL DEFAULT 0
            )
        """.trimIndent())
        db.execSQL("CREATE INDEX IF NOT EXISTS index_daily_plan_dayStartMs ON daily_plan(dayStartMs)")
        db.execSQL("CREATE INDEX IF NOT EXISTS index_daily_plan_isDone ON daily_plan(isDone)")
    }
}
```

**如果决定合并进 Migration_10_11**：直接把上面三行 `execSQL` 追加到 `Migration_10_11.kt` 的 `migrate` 方法末尾，不需要新建文件。

## 3. Repository 层

文件：`core/data/repository/DailyPlanRepository.kt`

```kotlin
interface DailyPlanRepository {
    fun observeToday(): Flow<List<DailyPlan>>
    fun observeTodayProgress(): Flow<Pair<Int, Int>>  // (done, total)，仅供 UI 显示

    suspend fun add(title: String, note: String? = null)
    suspend fun toggleDone(id: Long)
    suspend fun updateContent(id: Long, title: String, note: String?)
    suspend fun delete(id: Long)
    suspend fun reorder(ids: List<Long>)              // 批量更新 sortOrder
}

class DailyPlanRepositoryImpl @Inject constructor(
    private val dao: DailyPlanDao,
    private val clock: () -> Long = { System.currentTimeMillis() },
) : DailyPlanRepository {

    override fun observeToday(): Flow<List<DailyPlan>> =
        dao.observeByDay(todayStartLocalMs())

    override fun observeTodayProgress(): Flow<Pair<Int, Int>> {
        val day = todayStartLocalMs()
        return combine(dao.observeDoneCountByDay(day), dao.observeCountByDay(day)) { done, total ->
            done to total
        }
    }

    override suspend fun add(title: String, note: String?) {
        require(title.isNotBlank()) { "title must not be blank" }
        val now = clock()
        dao.insert(DailyPlan(
            title = title.trim().take(100),
            note = note?.trim()?.take(500)?.takeIf { it.isNotBlank() },
            dayStartMs = todayStartLocalMs(),
            createdAt = now,
            sortOrder = (now / 1000).toInt(),
        ))
    }

    override suspend fun toggleDone(id: Long) {
        val plan = dao.getById(id) ?: return
        val now = clock()
        dao.setDone(
            id = id,
            isDone = !plan.isDone,
            completedAt = if (!plan.isDone) now else null,
        )
    }

    override suspend fun updateContent(id: Long, title: String, note: String?) {
        require(title.isNotBlank()) { "title must not be blank" }
        dao.updateContent(id, title.trim().take(100), note?.trim()?.take(500)?.takeIf { it.isNotBlank() })
    }

    override suspend fun delete(id: Long) {
        val plan = dao.getById(id) ?: return
        dao.delete(plan)
    }

    override suspend fun reorder(ids: List<Long>) {
        ids.forEachIndexed { index, id ->
            val plan = dao.getById(id) ?: return@forEachIndexed
            dao.update(plan.copy(sortOrder = index))
        }
    }
}
```

DI 注入到 `RepositoryModule`：

```kotlin

```

## 5. 哲学红线（实现 AI 必读）

实现这个功能时**绝对不要**做以下事，会破坏 KHUP 的产品哲学：

- ❌ 不要在 TodayScreen 显示"今日计划完成 X / Y"摘要（哪怕在 MiniObservationCard 里）
- ❌ 不要让 AI / Generator 读取 `daily_plan` 表
- ❌ 不要在 prompt 里出现"用户的计划是 ..."上下文
- ❌ 不要做"未完成计划"的提醒推送
- ❌ 不要根据计划完成情况调整异常值建议
- ❌ 不要给计划加 "重要程度 / 优先级" 评分
- ❌ 不要做 "连续 N 天完成全部计划" 的 streak 显示
- ❌ 不要给 plan row 加"再加把劲"之类的鼓励文案

`daily_plan` 是用户的私有看板，**AI 完全不知情**。这是为了避免把"自我管理工具"和"算法干预工具"两者的边界混淆 —— 一旦 AI 开始评论用户的自定计划，KHUP 就退化成一个普通的 todo + 提醒 app 了。

## 7. 不在本补丁的工作

- 跨日计划（明天的计划）→ 下迭代
- 计划模板 / 复用 → 下迭代
- 时间块 / 日历视图 → 下迭代
- 计划与异常值建议的交互（互不知情，是设计选择）
- 计划完成数据进入"历史"tab 的趋势图 → 下迭代再讨论是否要做（涉及"是否衡量"边界）

本补丁只恢复**最简版每日计划功能**，让 KHUP 不再缺失一个原有的用户工具，仅此而已。
