### 2.3 AnomalySuggestionCard（最关键的组件）

**输入**：`AnomalySuggestion?` 状态

**4 种渲染状态**：

| State | 触发条件 | 显示 |
|---|---|---|
| Pending | 有 PENDING 建议 | 标题 + actionText + whyText + 三键按钮 |
| Generating | 用户刚提交 USER_REPORT，建议尚未生成 | "正在观察..." 占位（轻微 shimmer） |
| Empty | 无建议且非生成中 | 空态文案（见下） |
| Recent Accepted | 刚 accept，半天内 | 灰色淡化卡片 + "已采纳" 文字（不显示按钮） |

**Pending 状态布局**：

```
┌─────────────────────────────────────┐
│ 今日异常值                          │  ← 小号标签
│                                     │
│ 会议后低阻力恢复路径                │  ← title (h6)
│                                     │
│ 不要立刻回宿舍。先在楼下走 10 分钟， │  ← actionText (body1, 加粗)
│ 期间不刷手机，只观察一个最想逃避的    │
│ 念头。                              │
│                                     │
│ ─────                               │  ← 浅色分隔
│                                     │
│ 为什么这条                          │  ← 小号标签
│ 你现在不是单纯想休息，可能在寻找    │  ← whyText (body2, 浅色)
│ 低摩擦情绪缓冲。这个小偏离能让你    │
│ 重新获得选择权。                    │
│                                     │
│ 成本：低 · 收益：恢复主动选择        │  ← costLevel + expectedUpside (caption)
│                                     │
│ [接受]   [换一个]   [不适合]         │  ← 三键，高度 48dp
└─────────────────────────────────────┘
```

**只有 3 个按钮**（不要"完成"）。按钮顺序固定：接受 / 换一个 / 不适合。

**点击行为**：
- `[接受]` → `viewModel.acceptSuggestion(id)` → 卡片切换到 "Recent Accepted" 状态
- `[换一个]` → `viewModel.postponeSuggestion(id)` → 卡片立即切换到 "Generating" 状态，新建议来了再切回 Pending
- `[不适合]` → 弹一个**轻量** dialog（可选输入 reason，最多 100 字，可跳过）→ `viewModel.rejectSuggestion(id, reason)`

**Empty 文案**（统一）：

```
今天还没有可说的回归值信号。
要不要写一段你现在的处境？
```

**Generating 文案**：

```
正在观察...
```

加一个温和的 shimmer 或 progress dot。**绝对不要**写"AI 思考中"、"正在生成"、"加载中"等技术化字眼。

### 2.4 QuickCheckInCard

```
┌─────────────────────────────────────┐
│ 现在是什么处境？                    │  ← 小号标签
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 比如：刚开完会，平时回宿舍刷手机 │ │  ← TextField placeholder (浅色)
│ │                                 │ │  ← 多行，3 行高
│ └─────────────────────────────────┘ │
│                                     │
│             123 / 500    [提交 →]    │  ← 字数计数 + 提交按钮（右对齐）
└─────────────────────────────────────┘
```

**约束**：
- 最多 500 字符（前端硬截断 + Repository 防御性再截断）
- 提交时 trim 空白；blank 禁用提交按钮
- 提交后清空输入框
- 提交按钮禁用条件：blank 或 length > 500

**不要做的**：
- 不要弹 dialog 让用户输入；就在主页面输入，降低摩擦
- 不要"提交后有 toast 反馈"；卡片状态切换到 Generating 即足够反馈

┌─────────────────────────────────────┐

│                                     │

- 点 [查看通知 →] 跳到 **NotificationsScreen**（新增路由，见第 13 节）

- 检入次数：`EventDao` 今日 type=USER_REPORT 的 count

### 2.6 TodayViewModel

```kotlin
@HiltViewModel
class TodayViewModel @Inject constructor(
    private val behaviorReportRepository: BehaviorReportRepository,
    private val suggestionRepository: AnomalySuggestionRepository,
    private val eventDao: EventDao,
    private val anomalyDao: AttentionAnomalyDao,
    private val appSessionDao: AppSessionDao,
) : ViewModel() {

    data class UiState(
        val suggestionCardState: SuggestionCardState = SuggestionCardState.Loading,
        val checkInText: String = "",
        val isSubmitting: Boolean = false,
        val miniObservation: MiniObservation = MiniObservation(),
        val rejectDialogState: RejectDialogState? = null,
    )

    sealed class SuggestionCardState {
        object Loading : SuggestionCardState()
        object Empty : SuggestionCardState()
        object Generating : SuggestionCardState()       // 用户刚 submit 或 postpone
        data class Pending(val suggestion: AnomalySuggestion) : SuggestionCardState()
        data class RecentAccepted(val suggestion: AnomalySuggestion) : SuggestionCardState()
    }

    val uiState: StateFlow<UiState>

    fun onCheckInTextChange(text: String)
    fun submitCheckIn()                      // 调 behaviorReportRepository.submit；切到 Generating
    fun acceptSuggestion(id: Long)
    fun postponeSuggestion(id: Long)         // 切到 Generating
    fun openRejectDialog(id: Long)
    fun closeRejectDialog()
    fun confirmReject(reason: String?)
}
```

**Generating 状态判定**：
- 当 `submit` 或 `postpone` 后，`observeTodayPending()` 会先 emit null（旧已 POSTPONED 或暂无新建议），等几秒后 emit 新建议
- 给 ViewModel 加一个 `isWaitingForGeneration: Boolean` 内部状态：`submit` / `postpone` 时设 true，新建议到达时设 false（或超时 30s 后自动设 false 切回 Empty）
- Generating = `pendingSuggestion == null && isWaitingForGeneration`

### 3.3 [模式] PatternsTab

`AttentionAnomaly` 列表，按 `lastSeenAt` 倒序，过滤 `status = 'ACTIVE'`（默认）+ status switch 显示其他状态。

每行显示：
- title
- type (chip)
- frequency (出现 N 次)
- lastSeenAt (相对时间)

点击展开 `EvidenceListSheet`（modal bottom sheet）：
- 关联的 Event（最近 5 条 USER_REPORT，对该 anomalyKey 命中过的）
- 关联的 AppSession / 其他证据（暂时 MVP 不必全做，留 placeholder）

### 3.4 [建议] SuggestionsTab

`AnomalySuggestion` 按 status 分组，**默认显示**最近 30 天，按 createdAt 倒序：

```
ACCEPTED (12)         ← 折叠组
POSTPONED (3)
REJECTED (5)
PENDING (1)
```

每行显示：
- title (一行)
- actionText (两行截断)
- createdAt 相对时间
- 末尾 chip 显示 status

点击进入详情页（或 modal），显示：
- 完整 actionText
- whyText
- expectedUpside
- 关联的 pattern（可点击跳到 PatternsTab）
- 该 suggestion 的所有 UserFeedback 记录

### 3.5 [趋势] TrendsTab

最简版：
- 7 天 / 30 天切换
- 4 个折线图（共用 X 轴）：
  1. 每日检入次数（USER_REPORT count）
  2. 每日新建议数（AnomalySuggestion count）
  3. 接受率（ACCEPTED / 总数）
  4. 屏幕时间

**MVP 阶段**：
- 用 `androidx.compose.foundation.Canvas` 手画折线，**不引入图表库**（Vico/MPAndroidChart 等）
- 如果手画太麻烦，可以先用 `androidx.compose.material3` 的简化柱状图占位
- 数字必须显示，图可以糙
