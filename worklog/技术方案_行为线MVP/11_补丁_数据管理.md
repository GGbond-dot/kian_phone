# Module 11 — 补丁：数据管理（保留策略 + 清空 + 导出）

## 背景

当前 SettingsScreen 的 `DataCard` 三行全是 `Toast("下个版本实现")`：
- 导出全部数据
- 清空历史数据
- 数据保留策略

同时 `RetentionWorker` 只清理 `events` 表（3 天），其余表无限增长。
个人设备长期使用后 DB 会不断膨胀，且用户无法控制自己的数据。

本补丁分三部分：
- A. 扩展 RetentionWorker：覆盖所有表，策略合理
- B. 清空历史数据：确认对话框 + 真实清空
- C. 数据导出：JSON 文件导出到外部存储

## Part A：数据保留策略（自动清理）

### A.1 各表保留期规划

| 表 | 保留天数 | 理由 |
|---|---|---|
| `events` | 3 天 | 原始事件量大，已有 RetentionWorker |
| `app_sessions` | 90 天 | 趋势分析需要 30 天窗口，留 3 倍余量 |
| `derived_results` | 90 天 | 同 app_sessions |
| `hourly_summary` | 30 天 | 汇总已压缩，原始值无长期价值 |
| `trigger_tags` | 30 天 | 同 hourly_summary |
| `content_theme_tags` | 30 天 | 同 hourly_summary |
| `category_usage_cache` | 7 天 | 纯缓存，过期即无效 |
| `chat_message` | 90 天 | 用户对话历史，保留 3 个月 |
| `chat_session` | 90 天 | 随 chat_message 一起清理 |
| `anomaly_suggestion` | 180 天 | 历史建议有分析价值 |
| `attention_anomaly` | 180 天 | 模式记录，长期有价值 |
| `user_feedback` | 180 天 | 同 anomaly_suggestion |
| `daily_plan` | 30 天 | 每日计划过期后无价值 |
| `daily_review` | 90 天 | 日报告 |
| `action_logs` | 30 天 | 干预日志 |
| `user_memory` | 永久 | 量极小（最多 3 行），不清理 |

### A.3 改写 RetentionWorker

**文件**：`common/work/RetentionWorker.kt`

```kotlin
@HiltWorker
class RetentionWorker @AssistedInject constructor(
    @Assisted appContext: Context,
    @Assisted params: WorkerParameters,
    private val eventDao: EventDao,
    private val appSessionDao: AppSessionDao,
    private val derivedResultDao: DerivedResultDao,
    private val hourlySummaryDao: HourlySummaryDao,
    private val triggerTagDao: TriggerTagDao,
    private val contentThemeTagDao: ContentThemeTagDao,
    private val categoryUsageCacheDao: CategoryUsageCacheDao,
    private val chatMessageDao: ChatMessageDao,
    private val chatSessionDao: ChatSessionDao,
    private val anomalySuggestionDao: AnomalySuggestionDao,
    private val attentionAnomalyDao: AttentionAnomalyDao,
    private val userFeedbackDao: UserFeedbackDao,
    private val dailyPlanDao: DailyPlanDao,
    private val dailyReviewDao: DailyReviewDao,
    private val actionLogDao: ActionLogDao,
) : CoroutineWorker(appContext, params) {

    override suspend fun doWork(): Result {
        val now = System.currentTimeMillis()
        var failed = false

        // table → 保留天数
        val tasks = listOf(
            "events"               to { eventDao.deleteOlderThan(cutoff(now, 3)) },
            "app_sessions"         to { appSessionDao.deleteOlderThan(cutoff(now, 90)) },
            "derived_results"      to { derivedResultDao.deleteOlderThan(cutoff(now, 90)) },
            "hourly_summary"       to { hourlySummaryDao.deleteOlderThan(cutoff(now, 30)) },
            "trigger_tags"         to { triggerTagDao.deleteOlderThan(cutoff(now, 30)) },
            "content_theme_tags"   to { contentThemeTagDao.deleteOlderThan(cutoff(now, 30)) },
            "category_usage_cache" to { categoryUsageCacheDao.deleteOlderThan(cutoff(now, 7)) },
            "chat_message"         to { chatMessageDao.deleteOlderThan(cutoff(now, 90)) },
            "chat_session"         to { chatSessionDao.deleteOlderThan(cutoff(now, 90)) },
            "anomaly_suggestion"   to { anomalySuggestionDao.deleteOlderThan(cutoff(now, 180)) },
            "attention_anomaly"    to { attentionAnomalyDao.deleteOlderThan(cutoff(now, 180)) },
            "user_feedback"        to { userFeedbackDao.deleteOlderThan(cutoff(now, 180)) },
            "daily_plan"           to { dailyPlanDao.deleteOlderThan(cutoff(now, 30)) },
            "daily_review"         to { dailyReviewDao.deleteOlderThan(cutoff(now, 90)) },
            "action_logs"          to { actionLogDao.deleteOlderThan(cutoff(now, 30)) },
        )

        tasks.forEach { (table, action) ->
            runCatching { val n = action(); Log.i(TAG, "pruned $n rows from $table") }
                .onFailure { Log.w(TAG, "prune failed: $table", it); failed = true }
        }

        return if (failed) Result.retry() else Result.success()
    }

    private fun cutoff(now: Long, days: Long) = now - TimeUnit.DAYS.toMillis(days)

    companion object {
        private const val TAG = "KHUP/Retention"
        const val UNIQUE_NAME = "khup.retention"
    }
}
```

RetentionWorker 的调度保持不变（已在 WorkManager 注册，每天一次）。

### B.3 SettingsViewModel 接入

```kotlin
@HiltViewModel
class SettingsViewModel @Inject constructor(
    // ... 已有依赖 ...
    private val dataClearUseCase: DataClearUseCase,
) : ViewModel() {

    private val _clearState = MutableStateFlow<ClearState>(ClearState.Idle)
    val clearState: StateFlow<ClearState> = _clearState.asStateFlow()

    fun clearAllData() {
        viewModelScope.launch {
            _clearState.value = ClearState.InProgress
            runCatching { dataClearUseCase.clearAll() }
                .onSuccess { _clearState.value = ClearState.Done }
                .onFailure { _clearState.value = ClearState.Error(it.message ?: "未知错误") }
        }
    }

    fun resetClearState() { _clearState.value = ClearState.Idle }

    sealed interface ClearState {
        object Idle : ClearState
        object InProgress : ClearState
        object Done : ClearState
        data class Error(val message: String) : ClearState
    }
}
```

## Part C：数据导出

### C.1 范围

导出以下表到一个 JSON 文件，不导出系统级缓存表：

- `attention_anomaly`（回归值模式）
- `anomaly_suggestion`（历史建议）
- `user_feedback`（反馈记录）
- `chat_session` + `chat_message`（AI 对话）
- `daily_plan`（每日计划）
- `app_sessions` 最近 30 天（使用时长）

**不导出**：events（量太大）、category_usage_cache（纯缓存）、trigger_tags、content_theme_tags、hourly_summary（中间产物）

### C.2 DataExportUseCase

新建 `core/data/usecase/DataExportUseCase.kt`：

```kotlin
@Singleton
class DataExportUseCase @Inject constructor(
    private val attentionAnomalyDao: AttentionAnomalyDao,
    private val anomalySuggestionDao: AnomalySuggestionDao,
    private val userFeedbackDao: UserFeedbackDao,
    private val chatSessionDao: ChatSessionDao,
    private val chatMessageDao: ChatMessageDao,
    private val dailyPlanDao: DailyPlanDao,
    private val appSessionDao: AppSessionDao,
) {
    /**
     * 把数据聚合成 JSON 字符串，调用方负责写文件。
     * 在 IO 线程调用。
     */
    suspend fun buildExportJson(): String = withContext(Dispatchers.IO) {
        val now = System.currentTimeMillis()
        val thirtyDaysAgo = now - 30 * 86_400_000L

        val patterns = attentionAnomalyDao.getAll()
        val suggestions = anomalySuggestionDao.getAll()
        val feedbacks = userFeedbackDao.getAll()
        val sessions = chatSessionDao.getAll()
        val messages = chatMessageDao.getAll()
        val plans = dailyPlanDao.getAll()
        val sessions30d = appSessionDao.getSince(thirtyDaysAgo)

        // 用 kotlinx.serialization 或手动拼接
        buildString {
            appendLine("{")
            appendLine("  \"exportedAt\": $now,")
            appendLine("  \"appVersion\": \"khup-mvp\",")
            appendLine("  \"patterns\": ${Json.encodeToString(patterns)},")
            appendLine("  \"suggestions\": ${Json.encodeToString(suggestions)},")
            appendLine("  \"feedbacks\": ${Json.encodeToString(feedbacks)},")
            appendLine("  \"chatSessions\": ${Json.encodeToString(sessions)},")
            appendLine("  \"chatMessages\": ${Json.encodeToString(messages)},")
            appendLine("  \"dailyPlans\": ${Json.encodeToString(plans)},")
            appendLine("  \"appSessions30d\": ${Json.encodeToString(sessions30d)}")
            append("}")
        }
    }
}
```

**注意**：所有 entity 类需要加 `@Serializable`（或通过 `kotlinx.serialization` 插件自动处理）。如果当前没有引入 `kotlinx-serialization-json`，在 `build.gradle.kts` 加：

```kotlin
// app/build.gradle.kts

    id("org.jetbrains.kotlin.plugin.serialization") version "<kotlin_version>"
}
dependencies {
    implementation("org.jetbrains.kotlinx:kotlinx-serialization-json:1.7.3")
}
```

### C.4 SettingsViewModel 导出逻辑

```kotlin
private val _exportState = MutableStateFlow<ExportState>(ExportState.Idle)
val exportState: StateFlow<ExportState> = _exportState.asStateFlow()

fun exportData() {
    viewModelScope.launch {
        _exportState.value = ExportState.InProgress
        runCatching {
            val json = dataExportUseCase.buildExportJson()
            // 写到 Downloads 目录
            val fileName = "khup_export_${SimpleDateFormat("yyyyMMdd_HHmm", Locale.getDefault()).format(Date())}.json"
            val downloadsDir = Environment.getExternalStoragePublicDirectory(Environment.DIRECTORY_DOWNLOADS)
            val file = File(downloadsDir, fileName)
            file.writeText(json)
            file.absolutePath
        }
            .onSuccess { path -> _exportState.value = ExportState.Done(path) }
            .onFailure { _exportState.value = ExportState.Error(it.message ?: "导出失败") }
    }
}

sealed interface ExportState {
    object Idle : ExportState
    object InProgress : ExportState
    data class Done(val filePath: String) : ExportState
    data class Error(val message: String) : ExportState
}
```

在 `SettingsScreen` 用 `LaunchedEffect(exportState)` 显示结果 Toast：

```kotlin
LaunchedEffect(exportState) {
    when (exportState) {
        is SettingsViewModel.ExportState.Done ->
            Toast.makeText(context, "已导出到：${exportState.filePath}", Toast.LENGTH_LONG).show()
        is SettingsViewModel.ExportState.Error ->
            Toast.makeText(context, exportState.message, Toast.LENGTH_LONG).show()
        else -> Unit
    }
}
```

**权限**：Android 10+ 写 Downloads 不需要额外权限（`MediaStore` 方式）。但为了简单直接用 `File`，在 `AndroidManifest.xml` 确认已有：
```xml
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
    android:maxSdkVersion="28" />
```

Android 29+ 自动豁免，无需额外申请。

## 验收清单

### 数据保留

- [ ] RetentionWorker 运行后，logcat 出现每张表的 `pruned N rows` 日志
- [ ] 手动将 `category_usage_cache` 写入一条 8 天前的记录，运行 Worker 后该记录消失
- [ ] `user_memory` 表的行不被清理（最多 3 行，运行 Worker 后数量不变）

### 清空历史数据

- [ ] 点"清空历史数据"弹出确认对话框，文案包含"不可撤销"
- [ ] 取消后不执行任何删除
- [ ] 确认后出现进度指示，完成后 Toast"历史数据已清空"
- [ ] Database Inspector 验证所有业务表行数为 0（`user_memory` 不为 0）
- [ ] AI 设置（API Key / Base URL / Model）不受清空影响
- [ ] 清空后重启 App 不闪退，Today 页展示空态

### 数据导出

- [ ] 点"导出全部数据"后几秒内 Toast 显示文件路径
- [ ] 在手机文件管理里能找到该 JSON 文件，放在 Downloads 目录
- [ ] JSON 文件可被系统 JSON 查看器（或 PC 文本编辑器）打开，格式合法
- [ ] 导出的 JSON 包含 `patterns` / `suggestions` / `chatSessions` 字段
- [ ] JSON 中**不包含**原始 `events` 表内容（量太大）
