# Module 10 — 补丁：分层记忆架构

## 背景

Patch 09 的 `KhupContextSummarizer` 每次对话都实时查 DB 生成上下文，有两个问题：
1. 没有缓存，每次 session 都重复计算
2. 没有分层，不区分"今天的状态"和"这个月的稳定规律"

本补丁用**三层预压缩 + 读表**替代"每次查算"。核心思路：

```
错误做法：  每次对话 → 查 DB → 实时生成摘要 → 送进 prompt
正确做法：  定时后台 → 查 DB → 生成摘要 → 存表
           每次对话 → 读表 → 直接送进 prompt（毫秒级，免 LLM 调用）
```

**Token 总预算上限：按通道分级（`TOKEN_BUDGET_LOCAL = 350` / `TOKEN_BUDGET_API = 800`）**

> 实现：`KhupContextSummarizer.buildUserContext(tokenBudget)` 由 `AiChatViewModel` 按当前
> `AiProviderMode` 传入对应值；切换通道时自动重算。

| 层级 | 内容 | 存储上限（字符） | 更新方式 | 是否用 LLM |
|---|---|---|---|---|
| SHORT_TERM | 今日/本周即时状态 | ≤ 200 字符（≈ 120 token） | 每次读取时若过期则重算 | ❌ 纯 DB 聚合 |
| MEDIUM_TERM | 近 30 天规律压缩 | ≤ 160 字符（≈ 150 token） | WorkManager 每 7 天一次 | ✅ 一次压缩调用 |
| LONG_TERM | 跨月稳定身份特征 | ≤ 100 字符（≈ 80 token） | WorkManager 每 30 天一次 | ✅ 从中期提炼 |

三层合计最大约 350 token，API 通道 budget 为 800 时三层均可完整注入。

## 2. 短期记忆（SHORT_TERM）

**不调 LLM，纯 DB 聚合，生成时间 < 50ms。**

过期时间：1 小时（每次读取时判断 `expiresAt < now`，过期则重新生成并写表）。

### 2.1 生成逻辑

文件：`core/ai/memory/ShortTermMemoryBuilder.kt`

```kotlin
@Singleton
class ShortTermMemoryBuilder @Inject constructor(
    private val anomalyDao: AttentionAnomalyDao,
    private val suggestionDao: AnomalySuggestionDao,
    private val eventDao: EventDao,
) {
    suspend fun build(): UserMemory {
        val now = System.currentTimeMillis()
        val todayStart = todayStartLocalMs()
        val weekStart = now - 7 * 24 * 3600 * 1000L

        // 1. 今日检入次数
        val checkInCount = eventDao.countByTypeAndRange(
            EventType.USER_REPORT.name, todayStart, now
        )

        // 2. 本周活跃 pattern（取前 3 个，只要 title）
        val patterns = anomalyDao.getActivePatternsSince(weekStart, limit = 3)

        // 3. 本周建议 domain 分布
        val domainCounts = suggestionDao.getDomainCountsSince(weekStart)
            .sortedByDescending { it.count }
            .take(4)

        // 4. 本周建议接受/拒绝数
        val recentSuggestions = suggestionDao.getRecentWithStatus(weekStart)
        val accepted = recentSuggestions.count { it.status == "ACCEPTED" }
        val rejected = recentSuggestions.count { it.status == "REJECTED" }

        // 5. 组装（目标 80 字以内）
        val content = buildString {
            if (checkInCount > 0) append("今日检入${checkInCount}次。")
            if (patterns.isNotEmpty()) {
                append("活跃模式：${patterns.joinToString("、") { it.title }}。")
            }
            if (domainCounts.isNotEmpty()) {
                val dist = domainCounts.joinToString(" ") { "${it.suggestionDomain}×${it.count}" }
                append("本周建议领域：$dist。")
            }
            if (recentSuggestions.isNotEmpty()) {
                append("接受${accepted}拒绝${rejected}条。")
            }
        }.trim()

        val tokenEst = (content.length * 1.5).toInt()

        return UserMemory(
            type = "SHORT_TERM",
            content = content.take(200),         // 硬截断
            tokenEstimate = minOf(tokenEst, 120),
            generatedAt = now,
            expiresAt = now + 3600_000L,          // 1 小时
        )
    }
}
```

## 3. 中期记忆（MEDIUM_TERM）

**每 7 天由 WorkManager 调用一次 LLM 生成，存入 DB。**

费用：1 次 API 调用 / 周，成本极低。

### 3.1 压缩 Prompt

```kotlin
const val MEDIUM_TERM_COMPRESSION_PROMPT = """
你是一个数据压缩助手。
将以下用户行为统计数据压缩成一段不超过 80 字的中文摘要。

要求：
- 只保留最有规律的模式，去掉偶发数字
- 保留哪类建议被接受/拒绝的规律
- 保留用户拒绝的常见原因（如果有）
- 不加标题，不加列表符号，只输出一段话
- 绝对不超过 80 字

数据：
{data_json}
"""
```

### 3.2 WorkManager Job

文件：`common/work/WeeklyMemoryJob.kt`

```kotlin
@HiltWorker
class WeeklyMemoryJob @AssistedInject constructor(
    @Assisted context: Context,
    @Assisted params: WorkerParameters,
    private val llm: LlmEngine,
    private val userMemoryDao: UserMemoryDao,
    private val anomalyDao: AttentionAnomalyDao,
    private val suggestionDao: AnomalySuggestionDao,
    private val feedbackDao: UserFeedbackDao,
    private val redactor: PromptRedactor,
) : CoroutineWorker(context, params) {

    override suspend fun doWork(): Result {
        return try {
            val now = System.currentTimeMillis()
            val thirtyDaysAgo = now - 30 * 24 * 3600 * 1000L

            // 1. 查询 30 天数据
            val patterns = anomalyDao.getActivePatternsSince(thirtyDaysAgo, limit = 10)
            val suggestions = suggestionDao.getRecentWithStatus(thirtyDaysAgo)
            val feedbacks = feedbackDao.recentByTargetType("SUGGESTION", thirtyDaysAgo, 30)

            // 2. 聚合成结构化数据（不含原始文本）
            val dataJson = buildDataJson(patterns, suggestions, feedbacks)

            // 3. 调 LLM 压缩（用最低成本通道）
            val prompt = MEDIUM_TERM_COMPRESSION_PROMPT.replace("{data_json}", dataJson)
            val compressed = llm.generate(prompt).trim().take(160)   // 80字 × 2 bytes 硬截断

            // 4. 写入 DB
            userMemoryDao.upsert(
                UserMemory(
                    type = "MEDIUM_TERM",
                    content = compressed,
                    tokenEstimate = (compressed.length * 1.5).toInt().coerceAtMost(150),
                    generatedAt = now,
                    expiresAt = Long.MAX_VALUE,   // 由 WorkManager 定期替换，不自动过期
                )
            )
            Result.success()
        } catch (t: Throwable) {
            Log.w("WeeklyMemoryJob", "failed: ${t.javaClass.simpleName}")
            Result.retry()
        }
    }

    private fun buildDataJson(
        patterns: List<AttentionAnomaly>,
        suggestions: List<AnomalySuggestion>,
        feedbacks: List<UserFeedback>,
    ): String {
        // 只聚合，不转发原始 text（隐私保护）
        val patternSummary = patterns.map { mapOf("title" to it.title, "frequency" to it.frequency) }
        val domainStats = suggestions.groupBy { it.suggestionDomain }
            .mapValues { (_, list) ->
                mapOf("total" to list.size,
                      "accepted" to list.count { it.status == "ACCEPTED" },
                      "rejected" to list.count { it.status == "REJECTED" })
            }
        val rejectReasons = feedbacks
            .filter { it.feedbackType == "REJECT" && !it.reason.isNullOrBlank() }
            .mapNotNull { it.reason }
            .groupBy { it }
            .mapValues { it.value.size }
            .entries.sortedByDescending { it.value }
            .take(3)
            .map { it.key }

        return """
            {
              "active_patterns": $patternSummary,
              "suggestion_by_domain": $domainStats,
              "common_rejection_reasons": $rejectReasons
            }
        """.trimIndent()
    }
}
```

## 4. 长期记忆（LONG_TERM）

**每 30 天运行一次，把最近 3 条中期摘要再压缩为 60 字以内的稳定特征。**

文件：`common/work/MonthlyMemoryJob.kt`

### 4.1 压缩 Prompt

```kotlin
const val LONG_TERM_COMPRESSION_PROMPT = """
你是一个数据提炼助手。
将以下用户的三条周摘要提炼为一段不超过 50 字的稳定规律描述。

要求：
- 只保留跨周都出现的规律，忽略偶发模式
- 聚焦"在什么情境下容易回归"和"什么类型的建议有用/没用"
- 不加标题，不加列表，只输出一段话
- 绝对不超过 50 字

{summaries}
```

### 4.2 Job 实现（简化版）

```kotlin
@HiltWorker
class MonthlyMemoryJob @AssistedInject constructor(...) : CoroutineWorker(...) {

    override suspend fun doWork(): Result {
        return try {
            // 取最近 3 条中期记忆（包括当前和历史快照，如果有的话）
            // MVP 简化：只有 1 条 MEDIUM_TERM，生成 LONG_TERM 是对当前中期的进一步压缩
            val medium = userMemoryDao.getByType("MEDIUM_TERM") ?: return Result.success()

            val prompt = LONG_TERM_COMPRESSION_PROMPT
                .replace("{summaries}", medium.content)

            val compressed = llm.generate(prompt).trim().take(100)  // 50字上限

            userMemoryDao.upsert(
                UserMemory(
                    type = "LONG_TERM",
                    content = compressed,
                    tokenEstimate = (compressed.length * 1.5).toInt().coerceAtMost(80),
                    generatedAt = System.currentTimeMillis(),
                    expiresAt = Long.MAX_VALUE,
                )
            )
            Result.success()
        } catch (t: Throwable) {
            Result.retry()
        }
    }
}
```

## 5. KhupContextSummarizer 重写

**替换 Patch 09 的实现**，改为从表读取而不是实时计算。

文件：`core/ai/KhupContextSummarizer.kt`

```kotlin
@Singleton
class KhupContextSummarizer @Inject constructor(
    private val userMemoryDao: UserMemoryDao,
    private val shortTermBuilder: ShortTermMemoryBuilder,
) {
    companion object {
        const val TOKEN_BUDGET = 350
    }

    suspend fun buildUserContext(): String {
        val now = System.currentTimeMillis()

        // 1. 读三层记忆
        val short = getOrRefreshShortTerm(now)
        val medium = userMemoryDao.getByType("MEDIUM_TERM")
        val long = userMemoryDao.getByType("LONG_TERM")

        // 2. 按 token 预算拼装（SHORT 优先，LONG 最低优先）
        val result = StringBuilder()
        var usedTokens = 0

        short?.let {
            result.appendLine(it.content)
            usedTokens += it.tokenEstimate
        }

        medium?.let {
            if (usedTokens + it.tokenEstimate <= TOKEN_BUDGET) {
                result.appendLine(it.content)
                usedTokens += it.tokenEstimate
            }
        }

        long?.let {
            if (usedTokens + it.tokenEstimate <= TOKEN_BUDGET) {
                result.appendLine(it.content)
            }
        }

        return result.toString().trim()
    }

    private suspend fun getOrRefreshShortTerm(now: Long): UserMemory? {
        val cached = userMemoryDao.getByType("SHORT_TERM")
        return if (cached == null || cached.expiresAt < now) {
            // 过期或不存在，重新生成（纯 DB，不调 LLM）
            val fresh = try { shortTermBuilder.build() } catch (t: Throwable) { return cached }
            userMemoryDao.upsert(fresh)
            fresh
        } else {
            cached
        }
    }
}
```

## 7. Token 预算全局参考

实际送进 LLM 的 tokens 估算（中文 ≈ 1.5 token/字符）：

### 本地 Gemma 通道（`TOKEN_BUDGET_LOCAL = 350`）

```
系统提示词（塔勒布风格 + 推理框架）     ~400 token   固定
用户上下文摘要（三层合计上限）          ≤ 350 token   动态
当次用户输入                           ~100 token    变量
────────────────────────────────────────────────────────
对话前总开销                           ≤ 850 token
剩余用于对话历史（4k-8k context）      3k-7k token   ≈ 10-20 轮
```

### API 通道（`TOKEN_BUDGET_API = 800`）

```
系统提示词（塔勒布风格 + 推理框架）     ~400 token   固定
用户上下文摘要（三层均可完整注入）      ≤ 800 token   动态
当次用户输入                           ~100 token    变量
────────────────────────────────────────────────────────
对话前总开销                           ≤ 1300 token
剩余用于对话历史（128k context）        126k+ token   长文本讨论无压力
```

费用：每次对话约 0.001-0.003 美元，每周压缩调用约 0.0005 美元，几乎可以忽略。

**如需进一步节省**：把 `TOKEN_BUDGET_LOCAL` 从 350 改到 200，一行常量。

### WorkManager

- [ ] `WeeklyMemoryJob` 已注册（Android Studio → App Inspection → Background Task Inspector 能看到）

- [ ] `MEDIUM_TERM.content` ≤ 160 字符，`tokenEstimate` ≤ 150
- [ ] `MonthlyMemoryJob` 已注册

### Token 预算

- [ ] 本地 Gemma 通道下，`buildUserContext(350)` 返回字符串长度 ≤ 233 字符
  - 验证：`content.length * 1.5 <= 350` → `content.length <= 233`
- [ ] API 通道下，`buildUserContext(800)` 返回字符串长度 ≤ 533 字符
